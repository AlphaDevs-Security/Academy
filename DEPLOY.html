אתחל ודחוף לגיתהאב — פעם ראשונה

התיקייה שלך לא מחוברת לגיתהאב עדיין. הפקודות הבאות עושות את הכל בפעם אחת — מאתחלות, מחברות לריפו, ודוחפות:

# 1. נכנסים לתיקייה
cd "/Users/lior/Documents/Claude/Projects/AlphaDevs Public Academy"

# 2. מאתחלים גיט מקומי
git init
git branch -M main

# 3. מחברים לריפו בגיתהאב
git remote add origin https://github.com/AlphaDevs-Security/Academy.git

# 4. מוסיפים, יוצרים commit, ודוחפים
git add .
git commit -m "Full upgrade — courses + i18n + team access + kids course"
git push -u origin main --force
⚠️ למה --force? כי הריפו בגיתהאב כנראה כבר מכיל גרסה ישנה של הקורס. הפקודה הזו דורסת את מה שיש שם בגרסה החדשה והמלאה מהמחשב שלך. זה מה שאתה רוצה — המחשב הוא מקור האמת.
💡 אם הוא מבקש שם משתמש וסיסמה לגיתהאב: סיסמה רגילה לא תעבוד מאז 2021. צריך טוקן אישי (Personal Access Token):
1. כנס ל-github.com/settings/tokens
2. Generate new token (classic) → סמן את repo → Generate
3. העתק את הטוקן (פעם אחת בלבד יוצג!)
4. כשגיט שואל "Password" — הדבק את הטוקן במקום
בדיקה: אחרי הפקודות — תיכנס ל-github.com/AlphaDevs-Security/Academy. אתה אמור לראות את כל הקבצים: CNAME, Kids/, owner-login.html, וכל השאר.

הוסף רשומת DNS ב-Cloudflare

היכנס ל-Cloudflare, בחר את הדומיין alphadevs.co, ולחץ על "DNS" בתפריט הצדדי.

הוסף רשומה חדשה (Add record) עם הפרמטרים הבאים:

שדהערך
סוג (Type)CNAME
שם (Name)academy
יעד (Target)alphadevs-security.github.io
פרוקסי (Proxy status)DNS only (ענן אפור, לא כתום!)
TTLAuto
⚠️ קריטי — Proxy DNS only: חייב שהענן יהיה אפור (לא כתום). הענן הכתום שובר את ה-HTTPS של GitHub Pages. אם ראית כתום — לחץ עליו והוא יהפוך לאפור.
בדיקה: אחרי השמירה, תראה את הרשומה ברשימה. עוד כמה דקות (1-30) ה-DNS יתפשט לעולם.

חבר את הדומיין ב-GitHub Pages

היכנס ל-github.com/AlphaDevs-Security/Academy/settings/pages

בשדה "Custom domain" — הזן academy.alphadevs.co ולחץ Save.

GitHub ירוץ בדיקת DNS אוטומטית. אם הכל בסדר תראה ✅ ירוק.

סמן את Enforce HTTPS (אם זה לא מסומן — חכה כמה דקות שהאישור יוקצה ואז סמן).

💡 אם רואה אזהרה אדומה על DNS — חכה 5-15 דקות ורענן. ההתפשטות של DNS לוקחת זמן. בדרך כלל מהיר יותר ב-Cloudflare.
בדיקה: אחרי שהשמירה הסתיימה ויש ✅ ירוק — לחץ על הקישור באתר GitHub. הוא אמור לפתוח את academy.alphadevs.co.

🔧 בעיות נפוצות

"Connection refused" / "Site not reachable"

סבירות גבוהה — ה-DNS עוד לא התפשט. חכה 15-30 דקות ונסה שוב. אם אחרי שעה זה עדיין לא עובד — בדוק ש:

  1. הרשומה ב-Cloudflare באמת CNAME ולא A או AAAA
  2. הענן אפור (DNS only) ולא כתום (Proxied)
  3. הקובץ CNAME אכן עלה ל-GitHub (תוודא שאתה רואה אותו ברפו)

"Your site is not yet served from this domain"

GitHub עוד מבצע אימות. רענן את דף ההגדרות (Settings → Pages) ב-GitHub אחרי 5 דקות. אם זה לא מתעדכן — מחק את ה-Custom domain ושמור (כדי לאפס), ואז הכנס מחדש.

"Mixed content warning" / HTTPS לא עובד

חכה ל-Enforce HTTPS להיות זמין (יכול לקחת עד שעה). זה אישור שגיתהאב מקצה אוטומטית. עד שזה זמין — האתר יעבוד על HTTP בלבד.

"זה לא נכון מדויק לאן בקלאודפלייר אני נכנס"

השלבים ב-Cloudflare:

  1. היכנס ל-dash.cloudflare.com
  2. בחר את alphadevs.co מרשימת הדומיינים
  3. בתפריט הצדדי השמאלי — לחץ על DNS ואז Records
  4. בקצה העליון — לחץ + Add record
  5. מלא את הפרטים מהטבלה למעלה
  6. לחץ Save

"אני לא יודע אם הדומיין באמת בקלאודפלייר"

הרץ בטרמינל:

dig NS alphadevs.co +short

אם רואה שמות שמסתיימים ב-cloudflare.com — אתה ב-Cloudflare. אם רואה שמות אחרים (Google, GoDaddy, וכו׳) — תיגש לממשק שלהם, אבל הצעדים זהים בעיקרון: הוספת CNAME מ-academy ל-alphadevs-security.github.io.
