0קודם — איפה הקבצים מתארחים?

כדי שרימה ונטליה יוכלו לפתוח את הקישור, הקבצים צריכים להיות זמינים אונליין (לא רק במחשב שלך).

3 אפשרויות (מהקלה לקצינה):

  1. GitHub Pages (כבר עובד אצלך): תעלה את הגרסה החדשה ל-github.com/AlphaDevs-Security/Academy, ה-Pages יעדכן אוטומטית. הכתובת תהיה https://alphadevs-security.github.io/Academy/.
    אופן הפעולה: ב-Terminal:
    git add . && git commit -m "Team access" && git push
  2. Netlify Drop (פשוט בטירוף): netlify.com/drop → גרור את כל התיקייה → קבל URL חינמי תוך 30 שניות. אפשר אחרי זה לחבר דומיין.
  3. Custom domain (academy.alphadevs.co): הכי מקצועי — אבל דורש הגדרת DNS ב-Cloudflare. עיין בתכנית המאסטר לפרטים.
💡 אם הקבצים עדיין לא online: אפשר לשלוח להן את כל התיקייה כ-ZIP דרך WhatsApp/Drive, והן יפתחו את הקובץ בדפדפן (קליק כפול). אבל זה פחות נוח — חוויית הלינק הקליקבלית עדיפה.

🔧 בעיות נפוצות + פתרונות

"הקישור לא עובד"

1. וודא שהקבצים אכן online (GitHub Pages / Netlify / domain) — לא רק במחשב שלך.
2. בדוק שהכתובת נכונה. ה-base URL צריך להיות הכתובת אונליין שלך.
3. כל הקבצים צריכים להיות באותה תיקייה — assets/, days/, etc.

"היא מקבלת license modal במקום להיכנס"

זה אומר שהמפתח שלה לא מוכר במערכת. בדוק:
1. האם עדכנת את app.js עם המייל הנכון?
2. האם הקובץ app.js אכן עלה לשרת?
3. תני לה ידנית את המייל + מפתח, היא יכולה להזין ב-modal.

"היא רואה watermark על המסך"

אם זו רימה — זה תקין (היא Student, watermark מופיע).
אם זו נטליה — בדוק שהמייל שלה מוגדר נכון ב-MANAGERS ב-app.js. אם המייל לא תואם — היא נחשבת Student.

"איך אני יודע שמישהי לחצה על הקישור?"

כרגע אין מנגנון analytics מובנה. אופציות:
1. תשאל אותה ישירות.
2. בעתיד — נחבר Google Analytics או Plausible לעקוב אחרי כניסות.
